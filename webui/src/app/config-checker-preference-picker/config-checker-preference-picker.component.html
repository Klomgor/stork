Checkers list

The table lists available review checkers. Each of them is responsible for analyzing the daemon configuration to detect some particular issue. The table contains the checker states, and names. names, descriptions, selectors, and triggers.

The state indicates if the checker is enabled or disabled. The state may also be inherited from the global settings. In this case, the actual state is presented in the brackets. The state changes must be confirmed by clicking the submit button.

The selectors define the daemon groups for which the specific configuration checker runs.

The triggers designate the condition that fires the checker (e.g., when configuration or host reservation change has been detected, manually etc.).
