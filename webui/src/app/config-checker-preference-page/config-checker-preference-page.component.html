This page displays the configurations of the review checkers.

Stork can detect and report potential issues with the monitored servers' configurations. The generated reports contain detailed descriptions and suggestions to eliminate the most typical errors and misconfigurations. Each report is generated by a dedicated "checker" that validates a particular configuration aspect. The UI displays unique checker's names, so it is easy to identify which checker generated each report.

If any configuration reports are not relevant in your deployment, they can be disabled globally or for selected daemons. Any disabled checkers are not run, which improves both the clarity of the reports list and Stork's performance.

Use the list on this page to disable or enable the checkers globally (i.e. for all daemons). To override this choice for individual daemons, choose the desired App Name in the Services Status section of the DHCP dashboard, and then click the Checkers button in the Configuration Review Reports section at the bottom of the page.
